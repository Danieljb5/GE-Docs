Introduction

GameObjects

Scripting

All object hold a structure that allows user scripts to be added and run.

runFunction

The supplied function must match the correct signature - it must return void and it must accept a reference to a GameObject. An example function:

void exampleFunction(GameObject& parent)
{
    
}

The GameObject reference allows the function to access properties of the caller (e.g. move the parent object).

scriptObject

The engine supplies a base class for scripts, which contains several virtual functions that can be overriden:

onCreate - called by the constructor when the script is instantiated

onStart - called before the first call to onUpdate

onUpdate - called once every frame

onLateUpdate - called after onUpdate every frame

onDestroy - called by the destructor when the script is destroyed

An example class:

class ExampleClass : public ScriptObject
{
public:
    ExampleClass() : ScriptObject()
    {

    }

    void onStart() override
    {
        // initialisation code here
    }

    void onUpdate() override
    {
        // update code here
    }
};

Coroutines

Coroutines are functions that have a separate stack and can be paused and resumed, and are available in languages such as C#.

While C does have coroutines, they are not compatible with C++ exceptions, so I would only use them as a last resort.

C++ does not present one portable and safe way to set up a new stack, and change the stack pointer, so instead, coroutine emulation is used.

below is a copy of coroutines.hpp

#pragma once

#define cobegin     _loop = true; while(_loop) { _loop = false; switch(_state) { case 0:
#define coyield(x)  do { _state = __LINE__; return x; case __LINE__:; } while(0)
#define coreturn(x) do { _state = 0; return x; } while(0)
#define cofinish    default: _state = 0; _loop = true; }} return {}
#define cofinishv   default: _state = 0; _loop = true; }} return

template <typename Return, typename... _Args>
class Coroutine
{
public:
    Return operator()(_Args... _args)
    {
        return run(_args...);
    }

protected:
    virtual Return run(_Args... args) = 0;
    int _state = 0;
    bool _loop;
};

The class Coroutine can be inherited from to create a coroutine.

The macro cobegin is used at the start of a coroutine function

The macro cofinish is used at the end of a coroutine function

The macro cofinishv is used at the end of a coroutine function that returns void

The macro coyield(x) is used to pause the function and return a value

The macro coreturn(x) is used to return from the function and reset _state, allowing the function to be reused

below is an example of a coroutine:

#include <system.hpp>
#include <iostream>

class example_coroutine : public Coroutine<int>
{   // can accept further template args as coroutine parameters
protected:
    int run()
    {
        cobegin;

        for(i = 0; i < 10; i++)
        {
            coyield(i); // retume control to main and return i
        }

        cofinish;
    }

    int i = 0;
};

int main()
{
    example_coroutine ec;

    for(int i = 0; i < 10; i++)
    {
        std::cout << ec() << "\n";
        System::sleep(50_ms);
    }

    return 0;
}

In the above example, main calls the coroutine ec, and after increasing i by 1, ec returns control to main, where the application sleeps for 50ms. This repeats, until the coroutine finishes, allowing control to switch between main and ec without locks.

Note that all variables that need to persist between calling yield() and the function resuming must be member variables, and initialised directly after cobegin.

Also note that the coroutine implements the member function run. This is the virtual function provided by the base class, and it must match the coroutine's template. In this example, the template arguments are <int>, so it returns int and has no parameters, so run also must return int and have no parameters. If the coroutine had the parameters <void, int, float>, run would have a signature similar to the following:

void run(int {name}, float {name}) {...}
