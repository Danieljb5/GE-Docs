Object Tagging

An object's properties are defined by customisable tags - for example, to make the renderer draw an object, you add a renderableTag and a spriteTag, then supply a sprite to the spriteTag.

These tags are similar to components in an entity component system, but are much more lightweight and easier to manage, as each one holds the same number of variables, a void pointer (void*), and an integer id (uint8_t). Due to padding, there would be 6 bytes of unused space on each tag, so there are three extra variables (a uint32_t (4 bytes), a uint16_t (2 bytes) and a uint8_t (1 byte)) that can optionally hold additional data. This means that all tag objects are the same size (16 bytes), so they can be stored directly inside a GameObject without the need for pointers, unlike ECS, which would require polymorphism, so each object would need a pointer.

The following implementation would conform to the above specification:

struct Tag
{
    Tag(uint8_t id)
    {
        this->id = id;
    }

    void* data;
    struct
    {   // consider using __attribute__((packed)) to make this struct exactly 7 bytes
        uint32_t a;
        uint16_t b;
        uint8_t c;
    } unused;
    uint8_t id; // this should appear as const to other classes
    // consider implementing a const_proxy class using friends and operator overloads
};

Built In Tags

The table below contains values of the currently defined reserved tags

Tag NameDescription
transformTagAdds positional data to an object. Allows the game to insert the object into a spatial acceleration structure (e.g. a quadtree)
renderableTagMarks the object as renderable. data must point to a tag containing rendering data
linedataTagRendering data for a 2D line
rectdataTagRendering data for a 2D rect
tridataTagRendering data for a triangle
polydataTagRendering data for a polygon
circledataTagRendering data for a circle
spritedataTagRendering data for an image (sprite)
scriptfunctionTagRuns the script pointed to by data each update
scriptobjectTagAdds the script pointed to by data to be run
